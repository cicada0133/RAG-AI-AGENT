## Описание проекта

Проект представляет собой прототип системы глубокой переработки знаний и генерации мультимедийного контента. Платформа позволяет загружать массив документов и автоматически преобразовывать их в различные форматы:
- От коротких выжимок (саммари) до формализованных отчетов.
- Интерактивные презентации.
- Чат с источниками на базе RAG (Retrieval-Augmented Generation).

Система спроектирована с возможностью работы как в закрытом контуре с локальными моделями, так и с доступом в интернет, обеспечивая конфиденциальность данных. В рамках данного прототипа **LLM-модель (Qwen 2.5) разворачивалась на локальной машине и пробрасывалась в проект через туннель**, что позволило обеспечить независимость от сторонних облачных API и работу в изолированной среде.

## Основной функционал

*   **RAG-система**: Загрузка документов файлов и возможность задавать вопросы по их содержанию с указанием источников.
*   **Чат с документами**: Интерактивное общение с предоставленными данными.
*   **Составление формализованных отчетов**: Автоматическая генерация краткого резюме.
*   **Генерация презентаций**: Автоматическое создание PPTX по ключевым темам из источников.

## Запуск проекта

Проект разделен на Frontend и Backend части. Для удобного развертывания настроен `docker-compose`.

### Требования
- Docker и Docker Compose.
- Локально развернутая модель Qwen 2.5 (например, через Ollama). 
  *Примечание:* Модель может пробрасываться через предоставляемые скрипты туннелирования (например, `setup_tunnel.py` или `start_gpu_tunnel.bat`), если она работает на отдельной машине с GPU.

### Шаги установки

1. **Клонирование репозитория:**
   ```bash
   git clone <URL_вашего_репозитория>
   cd Central_invest
   ```

2. **Работа с локальной моделью (через Ollama):**
   - Установите [Ollama](https://ollama.com/) на ваш компьютер.
   - Запустите локальную модель в терминале:
     ```bash
     ollama run qwen2.5:7b
     ```
   - *Примечание:* По умолчанию Ollama слушает порт `11434`. Для работы из Docker пропишите переменную среды `OLLAMA_BASE_URL=http://host.docker.internal:11434` в `docker-compose.yml`. Если вы запускаете модель на отдельной машине/сервере с GPU (не локальной), используйте туннельные скрипты `setup_tunnel.py` / `start_gpu_tunnel.bat` для проброса портов через SSH.

3. **Запуск проекта через Docker Compose:**
   ```bash
   docker-compose up --build -d
   ```

4. **Доступ к сервисам:**
   - **Frontend:** доступен в браузере по адресу `http://localhost:3000`
   - **Backend API:** доступен по адресу `http://localhost:8000`

### Примечания по безопасности
